# 自动备份任务配置

## 概述

自动备份工作目录到 GitHub，每天凌晨3点执行。

## 文件位置

- **备份脚本**: `auto-backup.sh`
- **日志文件**: `logs/backup.log`
- **Token文件**: `config/github-token.md`

## Cron 配置

```bash
# 每天凌晨3点自动执行
0 3 * * * /home/lang/.openclaw/workspace/auto-backup.sh >> /home/lang/.openclaw/workspace/logs/backup.log 2>&1
```

## 备份流程

1. **检查Token** - 验证GitHub Token是否有效
2. **配置代理** - 通过v2rayA代理连接GitHub
3. **检查变更** - 是否有新文件或修改
4. **提交变更** - 自动生成提交信息
5. **推送到GitHub** - 推送到 `hklang/Openclaw`

## 手动执行

```bash
# 执行备份
bash /home/lang/.openclaw/workspace/auto-backup.sh

# 查看日志
cat /home/lang/.openclaw/workspace/logs/backup.log
```

## Token 失效处理

### 自动检测

脚本会自动检测Token是否有效。如果失效：
- 日志中会记录错误
- 停止备份
- 等待人工处理

### 手动更新Token

1. **获取新Token**
   - 访问：https://github.com/settings/tokens
   - 生成新的Personal Access Token
   - 勾选 `repo` 权限

2. **更新配置文件**
   ```bash
   # 编辑token文件
   nano /home/lang/.openclaw/workspace/config/github-token.md
   ```

3. **测试备份**
   ```bash
   bash /home/lang/.openclaw/workspace/auto-backup.sh
   ```

### 通知方式

Token失效时：
1. 日志中记录错误
2. 下次执行时会再次检测
3. **你需要手动更新Token**

## 常见问题

### Q: 备份失败了怎么办？

A: 
```bash
# 查看错误日志
cat /home/lang/.openclaw/workspace/logs/backup.log

# 手动执行查看详细信息
bash /home/lang/.openclaw/workspace/auto-backup.sh
```

### Q: v2rayA代理关闭了怎么办？

A: 确保v2rayA服务运行：
```bash
echo 'lang1982' | sudo -S snap services v2raya
```

### Q: 想修改备份时间？

A: 
```bash
# 编辑crontab
crontab -e

# 改为每天凌晨2点
0 2 * * * /home/lang/.openclaw/workspace/auto-backup.sh >> /home/lang/.openclaw/workspace/logs/backup.log 2>&1
```

### Q: 想暂停自动备份？

A: 
```bash
# 查看当前任务
crontab -l

# 删除任务（注释掉）
crontab -l | sed 's/^/# /' | crontab -

# 或直接清空
crontab -r
```

## 注意事项

1. **Token安全**: Token保存在 `config/github-token.md`，不要分享给他人
2. **网络要求**: 备份需要通过v2rayA代理连接GitHub
3. **日志查看**: 定期检查备份日志确保正常
